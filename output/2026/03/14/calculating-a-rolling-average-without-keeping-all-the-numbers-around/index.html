Calculating a rolling average without keeping all the numbers around

OK, if you know anything about maths you will read this and shake your head in dismay because it's just obvious and why did I even need to think about it? But... I was quite pleased to think this through, so I thought I'd write it down in case anybody else finds it useful.

So. Your job is to calculate the average of a set of numbers. This is not hard: the average value1 is the sum of the numbers divided by the count of the numbers. So the average of 2, 7, 8 is

2+7+8   17
----- = -- = 5.666…
  3      3

You know that bit already. But now I say: ok, here's a new number to add to the end of the list: 12. What's the average now?

One obvious way is to say, well, we know the numbers are 2, 7, 8, so I'll just re-do the calculation with 2, 7, 8, 12:

2+7+8+12   29
-------- = -- = 7.25
    4       4

But that means that you have to keep all the numbers around forever. What if the list of numbers is big? That seems like hassle. It would be nice if we were able to calculate the average just from knowing what the previous average was, wouldn't it? So you know 5.666… and you get told that the new number to add is 12: how do you get the new average of 7.25 just from that?

I needed to do this and didn't know how, so I sat and thought about it for a short time and it's actually not hard, which was good; I'd just never considered it. So, as long as you know the "current" average number, and you know how many numbers there were in the list, you're all good. Consider: we know the current average is 5.666… and that there were 3 numbers in the list that made that up. We don't need to know what the 3 numbers were (2, 7, 8) because if the three numbers we'd been given before were all 5.666… -- that is, the list of numbers up to now was 5.666…, 5.666…, 5.666… then when we took the average of that list, it would still be 5.666…, because that's what an average means. So, even though we don't know what the 3 numbers were, we can pretend that we do, and now say that our new list is 5.666…, 5.666…, 5.666…, 12. And we can work out the average of that! Or, put another way, we know that the average is

         sum of all numbers
5.666… = ------------------
          count of numbers

and because we can pretend that all the numbers were the average, we can say that the sum of all numbers is just the current average multiplied by the count:

         5.666…, 5.666…, "count" times
5.666… = -----------------------------
                   count

and so the new average is:

              (5.666…, 5.666…, "count" times) + 12
new average = -----------------------------
                       count + 1

which is

              (current average × count) + new number
new average = --------------------------------------
                          count + 1

So if you know the current average a, how many numbers there have been so far c, and the new number n, then the new a = ((a*c)+n)/(c+1). And you only need to keep a and c around, rather than the whole list of numbers.

Simple? Yes. But it was a useful thought for me that I could actually do this without having to keep the list around, which was handy in order to be able to calculate pi very inefficiently with coin flips on a BBC Micro, a perfectly reasonable thing to do on your Saturday evening in my opinion.
